Image Builder Wizard

Builds the command to run generate-bootstrap-image.py, which produces a bootstrap image staging bundle — a tar.gz you combine with the official Proxmox VE ISO to create bootable pre-install media that forges itself automatically on first boot.

This is a wizard, not a launcher. It generates the CLI command for you to copy and run in a terminal. No files are created here — the command must be run from the proxmox-bootstrap/ directory of the broodforge repo.

Generated Command

python3 generate-bootstrap-image.py --manifest forge-manifest.json
Run this from the proxmox-bootstrap/ directory. The command will print a single-use install passphrase — note it down before closing the terminal. It will not be recoverable from the bundle afterward.

After Running the Command

The command produces a bootstrap-image-{cell_id}-{timestamp}.tar.gz staging bundle. These are the manual steps to turn it into bootable media:

  1. Download the official Proxmox VE ISO for your target version from https://www.proxmox.com/en/downloads. broodforge does not download or redistribute Proxmox media.
  2. Extract the staging bundle and note the printed install passphrase:
    tar -xzf bootstrap-image-*.tar.gz
    The passphrase is also printed to the terminal when you ran the command above. It goes in answer.toml's root-password field and is not stored anywhere in the bundle.
  3. Combine with the Proxmox ISO using proxmox-auto-install-assistant (available from Proxmox):
    proxmox-auto-install-assistant prepare-iso <proxmox.iso> --fetch-from iso --answer-file iso-staging/answer.toml --output bootable.iso
    Or use the USB-preparation method described in Proxmox's "Automated Installation" documentation.
  4. Copy the forge package and first-boot hook into the ISO/USB so answer.toml's post-install hook can find them. See iso-staging/README.md inside the bundle for exact paths.
  5. Boot the target machine from the prepared media. Proxmox installs unattended, then broodforge-first-boot.service runs forge.sh automatically on first boot — bare metal → operational hatchery with no further interaction required.
  6. Rotate the install passphrase once forging completes. The forge package's phase-03 replaces it with a KeePass-managed credential, but you should confirm and discard/invalidate the original single-use value.
Supported baseline: The pre-install media path is an optional alternative to the standard FORGING.md workflow. If you already have Proxmox installed on the target host, continue from FORGING.md Step 1 instead — this wizard is only needed if you want a fully-automated bare-metal bootstrap.
